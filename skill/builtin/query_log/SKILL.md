# 查询日志 Skill

当需要排查线上问题时，使用此 Skill 查询线上日志系统。

## 可用工具

- `search_log`: 按关键字和时间范围搜索日志
- `get_error_logs`: 获取最近的错误日志
- `get_request_trace`: 根据 request_id 获取请求链路日志

## 使用场景

- 排查接口报错时，查询对应时间段的错误日志
- 通过 request_id 追踪完整请求链路
- 查看某服务最近的异常日志趋势

## 注意事项

- 所有查询均为只读操作
- 日志数据可能包含敏感信息，请仅用于故障排查
